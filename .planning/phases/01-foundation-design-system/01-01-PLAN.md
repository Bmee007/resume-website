Create `tests/phase1.spec.ts` with the following tests — all tagged `@smoke`:

**FOUND-01** — dev server page loads:
- Navigate to `/`, expect `<title>` to contain "Borina Keo"

**FOUND-02** — design tokens on `:root`:
- Evaluate `getComputedStyle(document.documentElement).getPropertyValue('--color-accent').trim()`
- Expect result to equal `#10B981` (or the rgb equivalent `rgb(16, 185, 129)`)
- Also check `--color-bg` resolves to `#0F172A`

**FOUND-03** — responsive layout at all three viewports:
- For each viewport (375, 768, 1280): navigate to `/`, expect no horizontal overflow
- Check: `document.body.scrollWidth <= window.innerWidth`

**FOUND-05** — config files exist (filesystem check via expect):
- Use `fs.existsSync` in a `test.use` fixture or Node.js assertion (not Playwright page) to confirm `astro.config.mjs`, `package.json`, and `.gitignore` exist in the project root. Use `test.expect(fs.existsSync('astro.config.mjs')).toBe(true)` etc.

**NAV-01** — nav is sticky:
- Navigate to `/`, scroll to bottom: `page.evaluate(() => window.scrollTo(0, document.body.scrollHeight))`
- Expect `#site-nav` to be visible (`.isVisible()`)
- Check nav CSS position: `page.locator('#site-nav').evaluate(el => getComputedStyle(el).position)` returns `"fixed"`

**NAV-02** — smooth scroll to section:
- Navigate to `/`, click `a[href="#about"]` in the desktop nav
- Expect `#about` to be in viewport: `page.locator('#about').isInViewport()`

**NAV-03** — active section highlighting:
- Navigate to `/`, scroll to `#skills` section: `page.locator('#skills').scrollIntoViewIfNeeded()`
- Wait 300ms for IntersectionObserver to fire
- Expect `a[href="#skills"].nav-link` to have class `active`

**NAV-04** — hamburger menu on mobile (375px viewport):
- Use `mobile` project; navigate to `/`
- Expect `#hamburger` to be visible, desktop nav links to be hidden (`hidden` class)
- Click `#hamburger`, expect `#mobile-menu` to be visible
- Click `a.mobile-nav-link[href="#about"]`, expect `#mobile-menu` to be hidden again

Tag each test with `@smoke` in the test name or via `test.tag`:
```typescript
test("FOUND-01 @smoke — page loads with correct title", async ({ page }) => { ... });
```
