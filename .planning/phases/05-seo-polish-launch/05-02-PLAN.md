  <name>Task 2: Extend BaseLayout.astro with OG tags, JSON-LD, canonical; create robots.txt</name>
  <files>src/layouts/BaseLayout.astro, public/robots.txt</files>
  <action>
    Update BaseLayout.astro Props interface to add optional OG/canonical fields:
    ```ts
    interface Props {
      title: string;
      description?: string;
      ogTitle?: string;
      ogDescription?: string;
      ogImage?: string;
      canonicalUrl?: string;
    }
    ```

    Destructure with defaults:
    ```ts
    const {
      title,
      description = "Borina Keo — ERP & WMS AI Integration Leader. 15+ years delivering AI integration across Microsoft Dynamics 365, Manhattan DFIO, and Azure OpenAI for Fortune 500 operations.",
      ogTitle = title,
      ogDescription = description,
      ogImage = "/og-image.png",
      canonicalUrl = Astro.url.href,
    } = Astro.props;

    const siteBase = Astro.site ?? new URL(Astro.request.url).origin;
    const ogImageAbsolute = new URL(ogImage, siteBase).href;

    const personJsonLd = {
      "@context": "https://schema.org",
      "@type": "Person",
      "name": "Borina Keo",
      "jobTitle": "ERP & WMS AI Integration Leader",
      "url": siteBase,
      "sameAs": [
        "https://www.linkedin.com/in/borinakeo",
        "https://github.com/Bmee007"
      ],
      "description": "AI integration leader specializing in Microsoft Dynamics 365, Manhattan DFIO, and Azure OpenAI for enterprise ERP and WMS modernization."
    };
    ```

    In `<head>`, after the existing tags, add:
    ```html
    <link rel="canonical" href={canonicalUrl} />

    <!-- Open Graph -->
    <meta property="og:type" content="website" />
    <meta property="og:url" content={canonicalUrl} />
    <meta property="og:title" content={ogTitle} />
    <meta property="og:description" content={ogDescription} />
    <meta property="og:image" content={ogImageAbsolute} />
    <meta property="og:image:width" content="1200" />
    <meta property="og:image:height" content="630" />

    <!-- Twitter Card -->
    <meta name="twitter:card" content="summary_large_image" />
    <meta name="twitter:title" content={ogTitle} />
    <meta name="twitter:description" content={ogDescription} />
    <meta name="twitter:image" content={ogImageAbsolute} />

    <!-- JSON-LD -->
    <script type="application/ld+json" set:html={JSON.stringify(personJsonLd)} />
    ```

    Update the default title in index.astro (if it passes a generic title) to:
    "Borina Keo — ERP & WMS AI Integration Leader | AI Consultant"
    This ensures the title contains "ERP" and "AI" for SEO-01.

    Also update the meta description call in index.astro to pass a keyword-rich description explicitly rather than relying on default, to confirm SEO-01 intent keywords are present.

    Create public/robots.txt:
    ```
    User-agent: *
    Allow: /

    Sitemap: https://borinakeo.vercel.app/sitemap-index.xml
    ```
    (Use the same URL as the site: value in astro.config.mjs — keep in sync.)

    CRITICAL: Do NOT use `color: #10B981` for any body paragraph or label text — emerald fails WCAG AA 4.5:1 for normal-weight text at body size. Accent use only (borders, glows, large headings).

    Test by running `npm run dev` and checking page source for og:image, JSON-LD script, and canonical tags.
