**Script block (inside `<script>`):**

Implement the Formspree AJAX submission handler from RESEARCH.md Pattern 2 exactly:
- `const FORMSPREE_ID = "YOUR_FORM_ID"` — placeholder constant
- Client validate on submit only (no on-blur): empty name → "Name is required", invalid email → "Please enter a valid email", empty message → "Message is required"
- On submit: disable button, set text to "Sending…"
- On `response.ok`: fade form out (opacity 0 over 300ms), then hide form + show success panel (fade in)
- On error: re-enable button, set text back to "Send Message", show error text under email field
- On network catch: show "Network error. Please try again." under message field, re-enable button
- Add `aria-live="polite"` to `#contact-success` element (already in HTML above)

The `fetch` call must use `headers: { Accept: "application/json" }` — without it Formspree returns a 302 redirect (CORS error per RESEARCH.md Pitfall 2).
